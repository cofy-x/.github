# Contributing to cofy-x Projects

Thanks for contributing to repositories under the **cofy-x** organization.

This file defines default contribution expectations. Individual repositories may
define additional rules in local documentation.

## Before You Start

Please read:

- [Code of Conduct](CODE_OF_CONDUCT.md)
- [Security Policy](SECURITY.md)
- [Support Guide](SUPPORT.md)

## Contribution Principles

- Keep pull requests focused on one concern.
- Prefer small, reviewable changes over large mixed updates.
- Update documentation when behavior or interfaces change.
- Add or update tests when introducing logic changes.

## Branch and Commit Style

- Branch from the repository default branch.
- Use [Conventional Commits](https://www.conventionalcommits.org/).

Examples:

```text
feat: add repository bootstrap command
fix: handle missing environment variable in startup
chore: align CI cache key strategy
```

## Pull Request Checklist

- Problem and solution are clearly described.
- Relevant tests or validation steps are included.
- Backward compatibility impacts are documented.
- No unrelated refactors are bundled into the same PR.

## Reporting Issues and Requests

- Bugs: use the Bug Report issue template.
- Feature requests: use the Feature Request issue template.
- Security issues: do not open public issues; follow [SECURITY.md](SECURITY.md).
