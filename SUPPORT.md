# Support

Use the appropriate channel so maintainers can respond effectively.

## Bug Reports

- Open a GitHub issue in the affected repository.
- Use the Bug Report template.
- Include reproducible steps, expected behavior, and actual behavior.

## Feature Requests

- Open a GitHub issue in the affected repository.
- Use the Feature Request template.
- Explain the problem, proposed solution, and tradeoffs.

## Usage Questions

- Prefer GitHub Discussions in the affected repository.
- If Discussions are not enabled, open an issue prefixed with `[Question]`.

## Security Issues

Do not use public issues for security reports.
Follow [SECURITY.md](SECURITY.md) for private disclosure.
